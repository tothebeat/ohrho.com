Favorite Python Modules

Wed 13 November 2013
By Nick Bennett

In Python.

tags: python

  • requests - Working with HTTP requests in a way that doesn't make me want to shove a stick in my eye.
  • pillow - I liked PIL, the Python Imaging Library, but I guess that wasn't good enough for some folks and so they made it better. I like the name too.
  • unicodecsv - Try working with international character sets in any kind of tabular data, that you need to work with in Python. You'll need this. And a machete.
  • beautifulsoup - An awesome html parsing module, that can use either lxml or html5lib for its engine.
  • future - Just for unicode_literals. It has a lot of imports that help prepare for Python 3, but this single import makes working with unicode strings so much less painful. No need to remember to tack on a 'u' before every string.
  • django - Hey, I have to like this because it's my job. It's pretty nice, don't get me wrong.
  • re - You can't avoid using this. Like it.
  • collections - Just for defaultdict. d = defaultdict(lambda: []) is just beautiful.
  • ipython - Nothing beats this shell.
  • selenium - This makes scraping easy.
  • sqlalchemy - I've barely scratched the surface with this, or even used it properly, but it has helped me vastly reduce the complexity of using a SQL database in the past.
  • xlrd/xlwt - Two modules for reading and writing Excel files, respectively. These can handle '.xls' files from Office 2003 and earlier, along with '.xlsx' files from Office 2007 and newer.
  • pelican - I have to represent for what created this blog.
